# 旅游路线推荐 QA 清单

## 目标

验证站间交通推荐是否符合真实旅行直觉：短距离省心，中距离考虑共享单车，长距离认真比较地铁/公交/打车。

## 样例路线

| 城市 | 起点 | 终点 | 重点观察 |
| --- | --- | --- | --- |
| 杭州 | 西湖断桥 | 灵隐寺 | 地铁/公交/打车谁更合理 |
| 杭州 | 杭州东站 | 西湖断桥 | 跨区公共交通是否优先 |
| 杭州 | 武林广场 | 河坊街 | 中短距离是否乱推荐打车 |

| 北京 | 天安门 | 颐和园 | 长距离是否推荐地铁 |

| 南京 | 夫子庙 | 中山陵 | 公共交通和打车对比 |

## 验收规则

- 800m 内优先步行，除非步行路线明显绕路。
- 1-4km 应出现共享单车候选；天气好、行李少时应合理。
- 3km 以上必须比较公共交通和打车。
- 地铁比打车慢 10-15 分钟以内时，可以优先地铁。
- 公交如果换乘多、步行多、耗时明显慢，不应作为主推荐。
- 每段站间交通应有推荐理由和至少 1 个可用备选。

## 高德/腾讯对比步骤

1. 腾讯默认配置跑一轮样例，截图或记录每段推荐。
2. 设置 `MAP_PROVIDER=amap` 和 `AMAP_WEB_SERVICE_KEY`（高德 Web 服务 Key；旧变量 `AMAP_MAP_KEY` / `GAODE_MAP_KEY` 仍兼容）。
3. 用同一批样例再跑一轮。
4. 对比每段的主推荐、耗时、换乘、步行距离。
5. 将明显不合理的路线加入回归样例。
