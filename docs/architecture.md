# 架构设计

## 产品定位

「树下小屋的工具箱」是一个轻量工具合集，小程序端以高频、低门槛、即时可用的工具为主。第一期工具选择「拼豆图纸生成器」，原因是它非常适合小程序交互：上传图片、调参数、生成结果、保存图片。

## 技术分层

### frontend

使用 uni-app + Vue3。当前优先开发微信小程序端，图片处理主要放在前端完成：

- `uni.chooseMedia` / `uni.chooseImage` 获取图片
- Canvas 对图片做居中裁切和像素采样
- 将采样色映射到拼豆色卡
- 页面渲染网格和颜色统计
- Canvas 导出图纸图片

这样第一版不需要上传用户图片，隐私和审核风险都更低。

### backend

使用 PHP 8.4 + Swoole + Hyperf。第一版后端只提供基础接口骨架：

- `/health`
- `/api/beads/palettes`
- `/api/beads/estimate`

后续可以把复杂能力放到后端：

- AI 抠图
- 图片卡通化或像素风增强
- 用户作品保存
- 色卡品牌管理
- 分享图生成

## 拼豆算法

当前算法流程（关键目标：放大清晰、无毛点、平坦区域颜色不混）：

1. 图片解码后粗缩到 ≤1200px 控制内存，按中心裁成正方形
2. 手写 box filter 面积平均降采样到网格尺寸（alpha 加权，透明格记为空格）
3. 可选：从四边缘 flood-fill 清除纯色/透明背景（只清边缘连通域）
4. LAB 空间 k-means 聚类降色到「最大颜色数」（固定种子，确定性输出）
5. 聚类中心用 ΔE2000 映射到拼豆色卡，多簇映射同色自然合并
6. 连通域去毛点：≤2 格的孤立色块并入邻居众数色，迭代至稳定
7. 生成网格图纸（四边行列号 + 格内色号 + 图例）和颜色用量统计

注意：必须**先聚类降色再匹配色卡**。逐像素独立匹配最近色会让照片的平坦区域在相近色号之间来回跳，产生噪点。

后续可以加入：

- 手动编辑单颗豆颜色
- 自动描边
- 手动调整裁切框
- 多品牌色卡
